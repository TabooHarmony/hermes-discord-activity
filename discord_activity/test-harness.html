URL Parameters

mock_discord=1 Enable mock Discord SDK for testing
session_id=xxx Override session ID
hermes_url=xxx Override Hermes API URL

Testing Flow

  1. Open Activity with ?mock_discord=1
  2. Use Hermes to launch a session: launch_activity tool
  3. Refresh Activity to see the session state
Note: For real Discord testing, you need:
  • Discord Application with Activities enabled
  • HTTPS hosting for the Activity URL
  • VITE_DISCORD_CLIENT_ID configured in build

Backend API Endpoints

GET  /api/plugins/discord-activities/state
GET  /api/plugins/discord-activities/state/{session_id}
POST /api/plugins/discord-activities/launch
POST /api/plugins/discord-activities/close
POST /api/plugins/discord-activities/event

GET  /api/plugins/discord-activities/health

Local Development

cd plugins/discord-activities/discord_activity
npm install
npm run dev     # Start dev server on port 3000
npm run build   # Build for production
Success: If you see "No Active Activity" message, the Activity shell is working correctly. Use Hermes tools to create a session.
